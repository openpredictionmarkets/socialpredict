## Executive Verdict

The May 24 frontend triage is mostly accurate as a baseline record, but it is no longer sufficient as the next-work queue by itself.

The May 26 embeddable-pages and market-sharing work has landed substantial backend, runtime, routing, CMS, and frontend metadata support. The next frontend upgrade should account for iframe and share-card behavior before starting broader platform work such as Redux, PWA, analytics, or a visual-system rewrite.

## Confirmed Iframe And Share Feature Work

The iframe/share notes should now be treated as active source material, not future speculation.

### Runtime Frame Policy

Confirmed:

- [../../../backend/internal/app/runtime/security.go](../../../backend/internal/app/runtime/security.go) reads `SECURITY_FRAME_ANCESTORS`.
- The default frame policy is deny: `frame-ancestors 'none'`.
- When an allowlist is configured, CSP receives the configured `frame-ancestors` value.
- `X-Frame-Options` is set to `DENY` for the default-deny case and omitted when a CSP frame-ancestor allowlist is configured.
- [../../../backend/internal/app/runtime/security_test.go](../../../backend/internal/app/runtime/security_test.go) covers the default deny and allowlist behavior.

Current status:

- Iframe behavior is an explicit runtime/operator policy.
- Deployed iframe behavior still needs release-environment verification.

- Non-public states return not-found behavior instead of leaking market-specific public metadata.

## Iframe-Specific Frontend Risks

Iframe support changes the frontend upgrade priorities because embedded pages are more sensitive to unsafe flows, route scope, and public metadata behavior.

Key risks:

| Risk | Why it matters | Direction |
| --- | --- | --- |
| Client-only metadata regression | Crawlers may not execute React. | Preserve backend share shell for `/markets/{id}`. |
| Accidental framing of sensitive routes | Admin, account, trading, and mutation flows are clickjacking-sensitive. | Keep iframe scope read-only unless explicitly approved. |
| Mixed API/auth call sites | Embedded routes may still call direct `fetch` or read tokens directly. | Migrate market detail and trading-adjacent surfaces first. |
| Raw errors in embedded UI | Embedded pages can expose confusing or sensitive runtime messages to partner sites. | Map public errors through safe copy. |
| Asset-name coupling | Share shell currently assumes stable built asset names. | Keep Vite asset naming stable or update share shell rendering strategy. |
| Deployment/proxy mismatch | Local behavior may differ from production header/proxy routing. | Verify headers and `/markets/{id}` routing in the deployed environment. |

## Do Not Start Yet

Do not start these before the active baseline and iframe/share verification are stable:

- Redux or a full global state rewrite
- PWA/offline/service-worker work
- product analytics or A/B testing
- browser APM/replay platform
- full visual-system rewrite
- strict bundle budgets
- broad Playwright/visual-regression infrastructure

These may become useful later, but they are not the next production-risk reduction step.

## Stop Conditions

Stop and re-triage if:

- frontend build is not reproducible from a clean install
- frontend tests cannot be added without dependency or runtime churn
- iframe policy requires framing authenticated or mutation routes
- backend session changes are required to make token handling meaningfully safer
- `/markets/{id}` share shell breaks normal SPA navigation
- production proxy behavior differs from local expectations
- a proposed dependency adds meaningful bundle size before the current size baseline is rechecked
